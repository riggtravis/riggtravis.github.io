Sudden Burst of Progress

Travis Rigg's Blogging and Development Extravaganza
2016-11-16
Debugging, HTML, JavaScript, Web Development

I haven’t had much to show for lately in my debugging. Today I found some very
interesting bugs.

What was going on?

None of my links were appearing upon loading the page. There were two factors
at play in this. The first was that the CSS framework I was using before was
squelching links. The second was that when I migrated I changed a name of an
element I was injecting content into.

When these links finally did appear, they weren’t links to anything. Something
was wrong with either my template or the navigation object I was creating. It
actually didn’t take me very long to discover that a double quote mark had been
left out. I never would have found this issue if I hadn’t already fixed the
other. It’s been a cycle of dependencies.

The most helpful aspect of this experience has been learning JavaScript
debugging strategies. I’m actually really happy about this.

What’s next?

I have created an infinite loop of page loading. It’s actually quite a
disastrous bug. The trouble is that with each successive page load, the
website’s memory usage increases until eventually my computer freezes. I
suspect that somewhere I have an event listener that is listening for an
element to load rather than for a click on this element.
